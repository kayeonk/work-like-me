# work-like-me

[English](./README.md) · **한국어**

> 내 AI 코딩 세션 로그로 **나만의 시스템 프롬프트**를 만든다 — 전부 로컬에서, 지시형으로.

AI한테 매번 같은 잔소리를 반복하고 있지 않나요. "시키지 않은 리팩터 하지 마", "결론부터 말해",
"완료라고 하기 전에 검증해". `work-like-me`는 로컬 Claude Code / Codex 세션 로그에서 **내 발화**를
읽어 이런 작업 스타일을 규칙으로 뽑아내고, 지시형 시스템 프롬프트로 컴파일해 전역 설정에 설치합니다.
한 번 해두면 **모든 프로젝트가 내가 실제로 일하는 방식을 그대로 물려받습니다.**

가상 캐릭터를 흉내 내는 "페르소나" 스킬이나 "대화 요약해줘" 같은 일회성 프롬프트와는 결이 다릅니다.

- **데이터로 만든다** — 지어낸 게 아니라 실제 세션에서 뽑고, 규칙마다 근거(등장 빈도)가 붙습니다.
- **로컬·비공개** — 네트워크 호출 0. 분석 전에 시크릿을 지웁니다.
- **지시형** — "~하라 / 하지 마라" 형태의 설치 가능한 프롬프트를 만듭니다. 설명문이 아니라.
- **직접 고른다** — 규칙은 레지스트리에 있고, 로컬 대시보드에서 켜고 끄거나 즐겨찾기할 수 있습니다.
- **알아서 자란다(선택)** — 대화가 쌓이면 훅이 새 신호를 모아 업데이트를 *제안*합니다.

![work-like-me 대시보드](docs/dashboard.ko.gif)

> 대시보드에서 규칙마다 카테고리·확신도·근거 수가 보입니다. 즐겨찾기(★)하거나 잠시 끄면 저장 전
> 상태로 표시되고, **변경 적용**을 눌러야 재컴파일·재설치됩니다. *(예시 페르소나라 실제 데이터 아님.)*

## 프라이버시

프라이버시가 이 도구의 전부라 먼저 짚습니다.

- **전부 로컬에서 돕니다.** 로그가 기기를 떠나지 않고, 어떤 텔레메트리도 없습니다.
- 샘플을 만들기 **전에** 시크릿·개인정보를 지웁니다 — 이메일, API 키(OpenAI/Google/AWS/Slack),
  GitHub 토큰, JWT, `Bearer` 헤더, `.env`식 시크릿 값, 긴 해시, IP, 그리고 **홈 경로(mac/Linux/Windows)**
  까지. 그래서 OS 사용자명도 안 샙니다.
- 회사·프로젝트 이름은 결과물에서 기본으로 익명화합니다.
- 다만 **완벽한 시크릿 제거기는 아닙니다.** 패턴 매칭이라 특이한 형식은 놓칠 수 있으니, **설치 전에
  컴파일된 프롬프트를 한 번 훑어보세요.**

## 준비물

- [Claude Code](https://claude.com/claude-code)
- Python 3
- (선택) Codex CLI 세션 로그

## ⚡ 설치

Claude Code와 Codex **둘 다** 같은 방식으로 됩니다.

**Claude Code** — 슬래시 명령이나 `/plugin` UI에서:
```text
/plugin marketplace add kayeonk/work-like-me
/plugin install wlm@kayeonk
```
설치 후 **Claude Code를 재시작**해야 스킬이 뜹니다. (`/reload-plugins`는 버전 따라 없을 수 있어요 —
새 세션이 제일 확실합니다. `/` 치면 `/wlm:create`가 보이면 성공.)

**Codex** — 터미널에서:
```bash
codex plugin marketplace add kayeonk/work-like-me
codex plugin add wlm@kayeonk
```
Codex는 세션 중 새로고침이 안 되니 **새 세션**에서 잡힙니다. 슬래시 명령은 없고 `$create`,
`/skills` 피커, `@wlm`, 또는 그냥 말로 부릅니다.

설치했으면 이렇게만 말하면 됩니다 — **"내 페르소나 만들어줘"**. 소스 선택 → 짧은 인터뷰 → 검토 →
설치 순으로 안내합니다.

> 로컬에서 개발하며 쓸 땐 `/plugin marketplace add ~/경로/work-like-me`로 잡으면 됩니다.
> 개인 데이터는 플러그인 밖 **`~/.work-like-me/`**에 있어서 업데이트해도 안 날아가고, 배포물엔 안 들어갑니다.

### 잘 안 될 때

- **설치했는데 스킬이 안 보임** → CLI를 재시작하세요(둘 다 세션 시작 때만 로드). 등록 확인은
  `/plugin marketplace list` (Claude) / `codex plugin marketplace list` (Codex).
- **`the name "wlm" is already taken`** → 예전 게 아직 깔려 있는 겁니다. 먼저 지우고
  (`/plugin uninstall wlm@<옛-마켓플레이스>`) 다시 설치하세요.
- **새 버전으로 올리기** → `/plugin marketplace update kayeonk` 후 재설치, 새 세션.
  Codex는 `codex plugin marketplace upgrade` + `codex plugin add …`.
- **`python3` 못 찾음** → `PATH`에 Python 3가 있어야 합니다.

## 스킬 네 개

| 스킬 | 하는 일 |
|------|---------|
| `create` | 로그에서 페르소나를 만들고 설치 |
| `dashboard` | 대시보드를 열어 규칙 켜기/끄기·즐겨찾기 후 적용 |
| `update` | 새로 쌓인 신호를 검토해 규칙 변경 제안 |
| `auto` | 자동 제안 켜기/끄기 |

페르소나는 **언어·직군을 안 가립니다.** 카테고리는 개발 전용으로 박아두지 않고 귀납 결과 그대로
씁니다. 풀스택처럼 한 사람이 FE/BE를 오가면, 따로 페르소나를 나누는 게 아니라 **한 페르소나 안의
상황별 규칙**("FE일 땐 …", "BE일 땐 …")으로 담습니다.

부르는 법:
- **Claude Code** — `/` 치고 `/wlm:create` 선택.
- **Codex** — `$create`, `/skills` 피커, `@wlm`.
- **공통** — 그냥 "내 페르소나 만들어줘"라고 말하기.

## 어떻게 동작하나

```
소스 탐지 → 추출(+레닥션) → 통계·샘플 → 표본 가드
→ LLM 귀납 → rules.json → 상황별 인터뷰
→ 컴파일·검토 → 설치(미리보기 → 적용)
→ 대시보드 관리 → 제안형 진화
```

로그 스캔·레닥션·통계 같은 기계적인 일은 스크립트가 하고, **실제 판단 모델링은 LLM이 레닥션된
샘플을 읽고** 합니다. 그래서 언어나 사람을 안 가립니다. 규칙은 `rules.json` 레지스트리에 모아뒀다가
프롬프트로 컴파일하고, 설치는 멱등적입니다(표시된 블록 안에만 쓰고, 처음 한 번 백업).

## 관리 & 진화

- **레지스트리** — 규칙(제목·지시문·확신도·근거·on/off·즐겨찾기)은 전부 `rules.json`에.
  `compile.py`가 켜진 규칙만 프롬프트로 만듭니다.
- **대시보드** — `python3 dashboard.py`면 로컬(127.0.0.1) 페이지가 열립니다. 클릭으로 즐겨찾기·끄기
  하고 한 번에 재설치. `rules.json`을 직접 고쳐도 됩니다.
- **제안형 업데이트** — **"wlm 업데이트"**라고 하면 실행할 때마다 최근 Claude·Codex 세션을 직접
  스캔해서(`capture.py --scan`) 바뀔 만한 규칙을 **제안만** 합니다. 승인 없이는 아무것도 안 바꿉니다.
- **자동 배지(Claude만)** — `auto` 스킬을 켜면 Claude `SessionEnd` 훅으로 신호를 조용히 모으다가
  충분히 쌓이면 대시보드에 알려줍니다. Codex엔 세션 종료 훅이 없어 이 편의만 Claude 전용이고,
  검토(`update`)는 양쪽 다 됩니다.

## 솔직한 한계

- **로그가 적으면 품질도 떨어집니다.** 발화 100건 안 되면 신뢰도 낮다고 경고합니다.
- **세션 로그는 "교정 지문"입니다.** 내가 반박·수정한 건 많이 남고, 조용히 넘어간 건 안 남습니다.
  그래서 결과는 완성품이 아니라 **손봐서 쓰는 초안**으로 보세요.
- 번들된 `analyze.py` 빈도 패턴은 한국어·개발 위주 **보조**일 뿐, 핵심 귀납은 LLM이 합니다.
- Claude(`~/.claude/CLAUDE.md`)·Codex(`~/.codex/AGENTS.md`) 전역 설정에 설치됩니다.
- 가끔 주입된 컨텍스트 줄이 샘플에 섞일 수 있습니다.
